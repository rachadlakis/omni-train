⚡ Features

Everything you need to fine-tune at scale

From a simple local experiment to multi-node SLURM clusters — OMNI-Train has you covered with a clean YAML-first API and a rich web UI.

🤖

Universal Model Support

Fine-tune CNNs, LLMs, VLMs, and YOLO detection models through one unified config. Switch model types with a single YAML field.

📈

LoRA & Quantization

Efficient fine-tuning with LoRA adapters and optional 4-bit / 8-bit quantization via bitsandbytes. Train billion-parameter models on consumer GPUs.

🌐

Distributed Training

Built-in DDP and FSDP2 support with activation checkpointing, mixed precision, gradient accumulation, and torch.compile.

🏭

Web UI Dashboard

Launch and monitor training jobs through a beautiful browser-based UI. Configure runs visually, view live logs, and manage a job queue.

💾

Smart Checkpointing

Automatic checkpoint management with best-model tracking, configurable save intervals, and seamless resume from any checkpoint.

📊

W&B Integration

First-class Weights & Biases support. Log loss curves, GPU metrics, learning rate schedules, and hyperparameters automatically.

☁

Multi-Source Data

Load datasets from HuggingFace Hub, S3/MinIO buckets, local files, torchvision, YOLO format, and COCO JSON — all via config.

🕒

Training Time Estimator

Accurate GPU-aware time estimates before you start. Supports all model types with realistic FLOPs-based calculations and GPU auto-detection.

🛠

SLURM Support

Ready-made SLURM batch scripts and a Python launcher for multi-node training on HPC clusters — no manual sbatch editing required.

🌐 Distributed Training

Scale from 1 GPU to 100+

Choose the right strategy for your model size and hardware with zero code changes — just update the config.

📀

Single GPU

Standard training with mixed precision and gradient accumulation. Perfect for prototyping and smaller models.

strategy: none
🔃

DDP

DistributedDataParallel across multiple GPUs or nodes. Each GPU holds a full model copy for maximum throughput.

strategy: ddp
⚡

FSDP2

Fully Sharded Data Parallel — shard model parameters, gradients and optimizer states. Train 7B+ models efficiently.

strategy: fsdp
All strategies support mixed precision, activation checkpointing, gradient clipping, and SLURM multi-node.
📚 How It Works

From config to trained model in minutes

A YAML-first workflow that's simple enough for beginners, powerful enough for researchers.

1

Write a YAML Config

Define your model type, dataset, distributed strategy, and hyperparameters in a single YAML file. Use one of 15+ included templates.

2

Launch Training

Run from the CLI or the Web UI. Single GPU, DDP, FSDP, or SLURM cluster — same command, different config.

3

Monitor in Real-time

Watch live training logs, loss curves, and GPU metrics via the Web UI dashboard or Weights & Biases.

4

Export Your Model

Checkpoints are saved automatically. The best model is tracked and ready for inference or further evaluation.

config/llm_lora.yaml 
model:
  type: llm
  name: meta-llama/Llama-2-7b-hf
  finetune_mode: lora
  lora_r: 16
  quantize: true
  quant_bits: 4

data:
  type: hf_dataset
  name: tatsu-lab/alpaca
  max_seq_len: 2048

distributed:
  strategy: ddp
  mixed_precision: true

training:
  epochs: 3
  batch_size: 8
  lr: 2e-4
  grad_accum_steps: 4
  lr_schedule: cosine
  wandb_enabled: true
  wandb_project: my-llm-finetune
terminal 
# Single GPU
python -m omni_train.train \
  --config configs/llm_lora.yaml

# Multi-GPU DDP
torchrun --nproc_per_node=4 \
  -m omni_train.train \
  --config configs/llm_lora.yaml

# CLI overrides
python -m omni_train.train \
  --config configs/llm_lora.yaml \
  --lr 1e-4 --epochs 5

# FSDP for large models
torchrun --nproc_per_node=8 \
  -m omni_train.train \
  --config configs/llm_fsdp.yaml

# Web UI
python -m omni_train.ui.app
# → http://localhost:8000

# SLURM cluster
sbatch scripts/slurm_train.sh \
  configs/llm_lora.yaml
💡 About

Built for researchers & practitioners

OMNI-Train was designed to eliminate the boilerplate that gets in the way of real research. Instead of writing the same training loop, distributed setup, and checkpoint logic for every project, you write a YAML config and get straight to the science.

  • ✓ Clean, registry-based architecture — easily extend with new model or data types
  • ✓ YAML-first with full CLI override support for sweep-friendly experimentation
  • ✓ Production-grade distributed training with DDP and FSDP2
  • ✓ Handles the full training lifecycle: data → train → checkpoint → resume
  • ✓ Web UI with live log streaming, job queuing, and GPU monitoring
