The Problem
Distributed training is not easy — and the hardware math forces you into it
1 / 3
🧠
Models Don't Fit
LLaMA-2 7B full fine-tune needs ≈84 GB of VRAM. Here's the math:
  • Weights — 7B × 2 bytes (bf16) = 14 GB
  • Gradients — same shape as weights = 14 GB
  • AdamW states — 2 moments × fp32 × 7B = 56 GB

An A100 80 GB cannot hold it alone — you need to shard across GPUs.

84 GBLLaMA-2 7B (bf16)
840 GBLLaMA-2 70B (bf16)
📚
Expertise Needed
  • torchrun principles — RANK, LOCAL_RANK, WORLD_SIZE, init_process_group, rendezvous, fault-tolerant elastic launch
  • NCCL Collectives — AllReduce, ReduceScatter, AllGather, Broadcast…
  • DDP — gradient all-reduce sync, bucket communication, no_sync context, rank-0 checkpoint writes
  • Libraries — PyTorch, Transformers, BitsAndBytes, PEFT, torchvision…
  • FSDP fully_shard — wrap policy, meta-device init, leaf-first order
  • DCP / DTensor — sharded save & load, broadcast_from_rank0
  • Mixed precision & dtypes — param / reduce / output dtype interaction
  • Saving / loading sharded models — rank-consistent state-dict reassembly
  • Prefetching — forward & backward prefetch depth, overlap with compute

— all before writing a single line of training code.

What Omni-Train Does For You
  • FSDP2 sharding — meta-device init, leaf-first wrap order, all-gather / reduce-scatter, mixed-precision policy
  • DDP — per-rank model load, all-reduce gradient sync, rank-0 checkpoint write
  • LoRA / QLoRA — PEFT adapter injection, 4-bit BitsAndBytes nf4/fp4, k-bit training prep, optional double quantization
  • PEFT + FSDP — LoRA adapters correctly wrapped inside FSDP sharding boundaries; gradients flow through sharded adapters without corruption
  • No setup code needed — GPU assignment, process coordination, and model loading across ranks are all taken care of automatically
  • Mixed precision — bf16 params, fp32 optimizer states, configurable reduce/output dtypes — no manual casting
  • W&B monitoring — opt-in loss curves, GPU memory snapshots, and per-rank diagnostics via all_gather_object
  • 20+ architectures — LLMs, Vision CNNs, YOLO, VLMs, embedding models — one model_type field
  • Single → multi-GPU — switch from solo to DDP to FSDP by changing one line in the YAML, zero code changes
Web UI / CLI + YAML File
Option 1 — CLI: edit config.yaml, then launch:
# single GPU python train.py --config config.yaml # multi-GPU (torchrun) bash launch.sh # SLURM cluster sbatch scripts/slurm_train.sh

Option 2 — Web UI: bash ui/launch_ui.sh → visual YAML editor. Toggle LoRA, pick strategy, click Launch — live logs stream in the browser.
