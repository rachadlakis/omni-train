Models Don't Fit
LLaMA-2 7B full fine-tune needs ≈96 GB of VRAM — weights + gradients + AdamW states. An A100 80 GB cannot hold it alone.
96 GBfor LLaMA-2 7B
560 GBfor LLaMA-2 70B

Silent Failure Modes
  • Missing env var → entire job hangs silently
  • Collective op inside if rank==0 → global deadlock
  • FSDP wrap order reversed → gradient corruption, no error
  • Wrong reduce_dtype → NaN loss, no warning
📚
Expertise Overhead
  • NCCL Collectives
  • FSDP Sharding Order
  • DCP Checkpointing
  • DTensor APIs
  • Mixed-Precision Policies
  • RANK vs LOCAL_RANK

— all before writing a single line of training code.

What It Handles Automatically
  • FSDP2 sharding — meta-device init, leaf-first wrap order, all-gather / reduce-scatter, mixed-precision policy
  • DDP — per-rank model load, all-reduce gradient sync, rank-0 checkpoint write
  • LoRA / QLoRA — PEFT adapter injection, 4-bit BitsAndBytes nf4/fp4, k-bit training prep, optional double quantization
  • Checkpointing — DCP API (broadcast_from_rank0) and DTensor API; safe resume with run-tag validation
  • Explicit prefetching — configurable forward/backward prefetch depth overlaps all-gather with compute
  • Gradient checkpointing — one flag discards and recomputes activations, saving 10–20 GB on long sequences
  • Rank-0 HF download + barrier — only rank 0 hits the network; all others wait then load the local seed
  • W&B monitoring — opt-in loss curves, GPU memory snapshots, and per-rank diagnostics via all_gather_object
  • 20+ architectures — LLMs, Vision CNNs, YOLO, VLMs, embedding models — one model_type field
