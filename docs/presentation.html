Why Distributed Training is Hard
Six traps that silently hang, corrupt, or OOM your job — straight from the code

⚡ Process Setup
Env Var Injection & Silent Hang
torchrun injects RANK, LOCAL_RANK, WORLD_SIZE into each process. One wrong value or missing var → entire job freezes with no error output.
# from distributed_utils.py rank = int(os.environ.get("RANK", "0")) local_rank = int(os.environ.get("LOCAL_RANK", "0")) dist.init_process_group(backend="nccl", device_id=torch.device(f"cuda:{local_rank}"))
🔗 Collective Ops
Ordering Deadlock
all_reduce, reduce_scatter, all_gather — every collective must be called by all ranks in the exact same order. One if rank == 0: around a collective → global hang, forever.
# ✗ Wrong — other ranks never call this if rank == 0: dist.all_reduce(tensor) # ← deadlock # ✓ Right — every rank calls it unconditionally dist.all_reduce(tensor)
🧱 FSDP Sharding
Layer Wrap Order
fully_shard(layer) must wrap each transformer block before fully_shard(model). Reverse the order → silent gradient corruption that trains but diverges.
# from distributed_utils.py → apply_fsdp() layers, _ = get_model_layers(model) for layer in layers: fully_shard(layer) # ← shard leaves first fully_shard(model) # ← then the root
🔒 Rank-0 Guard
Download Once + Barrier
HuggingFace downloads, wandb.init(), and checkpoint writes must only run on rank 0 — otherwise N processes race to download/write simultaneously, corrupting files.
# from train.py if rank == 0: wandb.init(project=args.wandb_project, ...) if dist.is_initialized(): dist.barrier() # ↑ all ranks wait here before proceeding
💾 Checkpointing
Shard Gathering (DCP API)
Naive torch.save(model.state_dict()) captures only rank 0's local shard. Must use DCP or DTensor API to reconstruct the full state dict — otherwise the checkpoint is unrestorable.
# from checkpoint.py → Checkpointer.save() set_model_state_dict( model, full_state_dict, options=StateDictOptions( full_state_dict=True, broadcast_from_rank0=True))
🎯 Mixed Precision
Three Dtypes, One Policy
MixedPrecisionPolicy has three independent dtypes. Mismatching reduce_dtype with param_dtype produces NaN gradients silently — no error, just divergence.
# from distributed_utils.py → apply_fsdp() mp_policy = MixedPrecisionPolicy( param_dtype = torch.bfloat16, reduce_dtype = torch.float32, # ← must differ output_dtype = torch.bfloat16)

Key Takeaways
  • AA consumer RTX 4090 (24 GB) cannot fine-tune LLaMA-2 7B with a full optimizer — QLoRA is the only option.
  • BFSDP divides every weight, gradient, and optimizer state shard across N GPUs — memory per GPU shrinks by exactly 1/N.
  • CGradient checkpointing trades ~30% speed for discarding and recomputing activations, saving another 10–20 GB on long sequences.
  • DWithout these tools, frontier model research is gated behind expensive hardware that most teams do not have.

Full model on every GPU — gradients synchronized via all-reduce after each backward

From the code — apply_ddp()
# Every rank loads its own full model copy model = AutoModelForCausalLM.from_pretrained( args.model_name, dtype=torch.bfloat16, low_cpu_mem_usage=True, ) model = model.to(device) # DDP hook: fires all-reduce on each param bucket after backward model = DDP( model, device_ids=[local_rank], find_unused_parameters=False, )
  • find_unused_parameters=False skips graph scanning for unlinked params — required when all parameters participate every forward pass.
  • PEFT (LoRA) and QLoRA work with DDP — adapters load normally, frozen base weights still replicated on each GPU.
  • Checkpoint: rank 0 saves model.module.state_dict() — a single portable file, no shard gathering needed.
💾 Memory
Full replica per GPU — no sharding
Each GPU holds the complete model weights, gradients, and optimizer states. Memory per GPU does not shrink with more GPUs — throughput scales, not capacity.
📡 Communication
Ring All-Reduce per parameter bucket
DDP groups parameters into buckets and launches one all_reduce per bucket. Gradients are averaged across all ranks before the optimizer step. Cost ≈ 2(N−1)/N × params.
✅ Best For
Models that fit in a single GPU
Use DDP when full model + optimizer fits in one card's VRAM. Lower implementation complexity than FSDP, no all-gather overhead during forward, simpler checkpointing.

💾 Memory
Weights ÷ N · Grads ÷ N · Optim ÷ N
7B model on 4 GPUs: 14 GB weights → 3.5 GB per card. Gradients and AdamW states are also sharded — the full 96 GB budget shrinks to ≈24 GB total across all GPUs.
📡 Communication
All-Gather + Reduce-Scatter
Forward: all-gather materializes full layer weights per rank, then discards after the pass. Backward: reduce-scatter keeps only the local gradient shard. Explicit prefetching overlaps communication with compute.
⚡ Mixed Precision Policy
Three independent dtypes
param_dtype=bfloat16 (compute) · reduce_dtype=float32 (stable grad reduction) · output_dtype=bfloat16. Set via MixedPrecisionPolicy passed to fully_shard().

The Hard Things This Project Saves You From
Concepts users never need to understand — but the framework handles correctly
6 / 9
NCCL Collectives init_process_group(backend="nccl", device_id=cuda:N) auto-maps to hardware. DDP uses all-reduce; FSDP uses reduce-scatter (keep shard) + all-gather (fetch on demand). Every collective must be called by all ranks in the same order or the job hangs.
FSDP Layer Sharding get_model_layers() probes the architecture and calls fully_shard(layer) per transformer block before sharding the root — wrong order silently corrupts gradients.
Explicit Prefetching set_modules_to_forward/backward_prefetch overlaps the all-gather of layer N+1's weights with layer N's compute — hides NVLink latency. Set via prefetch.forward: 2.
DCP & DTensor Checkpointing DCP: get_model_state_dict(cpu_offload=True) gathers all shards to rank 0; set_model_state_dict(broadcast_from_rank0=True) re-shards on load. DTensor: distribute_tensor(param, mesh) manually re-shards each parameter by name.
Mixed Precision Policy Three independent dtype fields: param_dtype (bf16), reduce_dtype (fp32 for stable all-reduce), output_dtype (bf16). Any mismatch causes NaN loss or silent numerical degradation.
Global vs Local Rank RANK = global index (0…world_size−1) for barriers & logging. LOCAL_RANK = per-node index used to bind cuda:N. Confusing them maps every process to cuda:0 — silent parallelism failure.
ZeRO Stages ZeRO (Zero Redundancy Optimizer) removes the redundant copies of model state that DDP keeps on every GPU. Stage 1 shards optimizer states only. Stage 2 also shards gradients. Stage 3 shards weights too — equivalent to FSDP. Without understanding which stage is active, you cannot reason about memory usage, communication volume, or why a seemingly correct config still OOMs.
