// 02 — WHAT IT NORMALLY TAKES

What distributed training requires.

Weeks of plumbing — and a dozen ways to fail silently — before training step 1.

Launch & comms
Process orchestration
  • torchrun — RANK, WORLD_SIZE, rendezvous, elastic launch
  • NCCL — all-reduce, reduce-scatter, all-gather, barrier
  • ZeRO 1/2/3 — how FSDP maps to optimizer + grad sharding
Sharding & memory
Getting weights to fit
  • fully_shard — wrap policy, meta-device init, leaf order
  • Mixed precision — bf16 params vs fp32 optimizer states
  • Prefetch + grad checkpoint
Persistence & traps
Saving — and silent death
  • DCP / DTensor — sharded save, rank-0 reassembly
  • Library quirks — Transformers · PEFT · BitsAndBytes
  • Silent failures — deadlock, NaN loss, grad corruption — no stack trace

// 04 — CHOOSE YOUR STRATEGY

Three strategies. Different problems.

solo, DDP and FSDP solve different bottlenecks — a single card, raw throughput, or model size. OMNI-Train runs all three from the same training code; you pick the right one for the job.

solo1 GPU
You have one GPU
No torchrun, no process group — the prototyping path for small models.
ddpcompute-bound
Everything fits — you want speed
Params + grads + optimizer states fit on one card, so every GPU holds a full replica; all-reduce syncs gradients each step. Throughput scales as you add GPUs.
fsdpmemory-bound
Params + grads + optimizer states won't fit
When the full footprint blows past one card — OOM — FSDP shards all three across ranks, ZeRO-3 style.

// 06 — EFFICIENT FINE-TUNING

LoRA, QLoRA, 4/8-bit — with guardrails.

Every parameter-efficient path is wired correctly — and the impossible ones are blocked before the GPU ever sees them.

Supported methods
  • LoRA — PEFT adapter injection, all-linear or named targets
  • QLoRA — quantized base + LoRA: 4-bit nf4/fp4 or 8-bit LLM.int8(), double-quant, k-bit prep
  • PEFT + FSDP — adapters wrapped inside shard boundaries; grads flow uncorrupted
Guardrails in build_args()
  • qlora → auto-enables quantization — 4-bit NF4 or 8-bit LLM.int8()
  • Any quantization requires PEFT — direct 4/8-bit training NaNs from step 1
  • fsdp + quantization → hard error (sharding can't touch quantized layers)
✓caught at config time, before allocation
// 07 — IMPLEMENTATION DETAILS

The edge cases, handled correctly.

The details that silently break hand-rolled training loops — solved once, reused everywhere.

Initialization
FSDP2 meta-device
Build on meta (zero memory), shard, then populate via 3 paths: resume, fresh-HF seed, or random init.
Persistence
DCP + DTensor I/O
Two sharded checkpoint APIs — rank-0 broadcast or manual distribute_tensor gather — with timestamped, resumable folders.
Correctness
RoPE buffer fix
Non-persistent inv_freq stays on meta after load — OMNI recomputes it from the formula instead of zeroing (which silently corrupts positions).
Mixed precision, managed
bf16 params, fp32 optimizer states, configurable reduce/output dtypes — no manual casting.
Layer auto-detection
Probes model.layers, transformer.h, encoder/decoder stacks to find blocks for per-layer sharding.
Per-rank diagnostics
Rank-0 banners + loss bars; W&B opt-in with GPU memory snapshots via all_gather_object.
